Plain-Language Explainer

The scout is an experiment picker.

It reads public demo rows, keeps each source caveat visible, and asks which example best illustrates the next scientific question.

Loop

What happens in one pass?

1. Read a row

Example: coumarin in water, or 6-methylcoumarin with CB8.

2. Name the organizing idea

Water heterogeneity, host confinement, micelle/cage organization, or schema mechanics.

3. Keep the caveat

The output says what the row does and does not prove.

4. Order the rows

The order is illustrative triage, not model accuracy.

5. Write explanation cards

Each card explains the source, why it matters, and the next check.

6. Decide the next action

The scout points to the next source check or experiment instead of pretending uncertainty is gone.

Examples

How a reviewer should read the rows

6-methylcoumarin + CB8

Shows a public host-water comparison. The crystal metric is useful as a prior, but it is not a direct picture of water-phase geometry.

Coumarin in water

Shows direct aqueous photodimerization context. Keep the row qualitative unless quantitative values are checked against the paper.

Trans-cinnamic acid host rows

Show public host-templating precedents. They are family-level examples, not derivative-specific predictions.

One Sentence

The scout turns public chemistry examples into a transparent next-question engine: what does this row suggest, what is still uncertain, and what should be checked next?

Technical Translation

Where the code fits

The public script scripts/run_public_scout.py loads data/public_demo_cases.csv, sorts rows by public-use status and row class, then writes a ranked CSV, Markdown explanation cards, and an SVG overview.
